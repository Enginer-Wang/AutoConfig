📖 使用文档

欢迎使用 Autoconfig 静态网页托管平台。以下文档将帮助你快速上手。

🚀 快速开始

第一步：准备你的项目文件

将你的静态网页项目打包成 .zip 文件。项目中至少应该包含一个 index.html 文件作为入口页面。

支持的项目类型：

  • 纯 HTML/CSS/JS 静态网页
  • React 构建产物（npm run build 后的 build/ 或 dist/ 目录）
  • Vue 构建产物（npm run build 后的 dist/ 目录）
  • Next.js 静态导出（next export 后的 out/ 目录）
  • 任何静态网站生成器的输出

第二步：上传部署

登录后进入控制台，将 ZIP 文件拖拽到上传区域，或点击选择文件。

填写项目信息：

  • 项目名称 - 项目的显示名称
  • 项目路径 (slug) - 访问地址中的路径部分，如 my-project
  • 项目描述 - 可选，项目的简短介绍
  • 是否公开 - 勾选后项目将出现在社区广场

第三步：访问你的站点

部署成功后，你将获得一个专属链接：

https://你的用户名.autoconfig.dev/项目路径

在本地开发环境中，访问地址为：

http://localhost:3000/site/你的用户名/项目路径

📂 项目管理

重新部署

上传同名 slug 的项目时，系统会自动替换原有文件，实现无缝更新。

删除项目

在控制台点击项目卡片上的「删除」按钮。注意：此操作不可撤销，所有文件将被永久删除。

公开/私密切换

每个项目可以在公开和私密之间切换：

  • 公开 - 项目会出现在社区广场，任何人都可以浏览
  • 私密 - 只有知道链接的人才能访问

🌐 社区广场

社区广场展示所有公开的项目。你可以：

  • 浏览其他开发者的作品
  • 搜索感兴趣的项目
  • 获取灵感和学习

⚙️ 技术规格

  • 单个项目最大 50MB
  • 支持 HTML、CSS、JavaScript、图片等所有静态资源
  • 自动识别并跳过 ZIP 中的根目录包裹
  • SPA 单页应用路由兼容（自动 fallback 到 index.html）

❓ 常见问题

为什么上传后页面是空白的？

请确保你的 ZIP 文件中包含 index.html 文件。如果你的 HTML 文件在子目录中，请确保 ZIP 的结构正确。

如何更新已部署的项目？

使用相同的项目路径 (slug) 重新上传即可。系统会自动替换旧文件。

项目链接会过期吗？

不会。只要你不手动删除项目，链接将一直有效。
